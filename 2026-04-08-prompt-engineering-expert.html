为什么值得学？

提示词工程是 AI 时代的底层操作系统。你用的每一个 AI 工具、每一个 Agent、每一个 Skill——背后都是提示词在驱动。写好提示词的人和写不好的人，得到的结果可能差 10 倍。

  • 必学 所有 AI 用户——提示词是你和 AI 沟通的唯一语言，学好它等于给自己加 10x 杠杆
  • 适用 Skill/Agent 开发者——System Prompt 设计、行为约束、测试框架全在这
  • 好学 纯方法论，不需要编程基础，看例子就能上手
  • 进阶 已有提示词经验的人——反模式识别和调试工作流能帮你突破瓶颈
核心理念：四大原则
🎯
Clarity 清晰
显式目标、精准语言、具体示例、逻辑结构。说"我要什么"，不要让 AI 猜。
✂️
Conciseness 简洁
聚焦内容、消除冗余、渐进式披露、Token 效率。每个字都要有用。
📐
Consistency 一致
定义约束、指定格式、清晰准则、可复现结果。同一个 Prompt 出来的结果应该稳定。
🧩
Completeness 完备
充分上下文、边界情况、成功标准、错误处理。别假设 AI 什么都懂。
七大提示词技术（可折叠）

点击展开每个技术的详细说明和代码示例 👇

1
Chain-of-Thought 链式思考
▼

让 AI 一步一步推理，而不是直接给答案。适用于复杂推理、多步骤问题、需要可验证逻辑的场景。

// 基本结构 Let's think through this step by step: Step 1: [First logical step] Step 2: [Second logical step] Step 3: [Third logical step] Therefore: [Conclusion]

关键洞察：CoT 不只是让输出好看——它实际上改变了 AI 的推理路径，就像你考试时把演算过程写出来，正确率会比直接写答案高很多。

2
Few-Shot Learning 少样本学习
▼

用例子教 AI 你要什么，比描述规则更直接。1-shot 适合简单任务，2-shot 适合中等复杂度，Multi-shot 适合复杂模式。

// 情感分类 Few-Shot Example 1: "Great product!" → Positive Example 2: "Doesn't work well" → Negative Example 3: "It's fine" → Neutral Now classify: "Not bad" →

最佳实践：用多样化的例子、包含边界情况、按复杂度排序。例子选得好，效果比写一页规则还好。

3
XML Tags 结构化标签
▼

用 XML 标签给 Prompt 加结构，让 AI 知道哪部分是目标、哪部分是约束、哪部分是格式要求。对复杂任务的输出一致性提升巨大。

<task> <objective>What to accomplish</objective> <constraints>Limitations and rules</constraints> <format>Expected output format</format> </task>

为什么有效：XML 标签给 AI 提供了"语义边界"，它能更精准地理解每段文字的角色。对 Claude 特别有效。

4
Role-Based Prompting 角色扮演
▼

给 AI 一个具体身份——不是笼统的"当个专家"，而是"你是一个有10年经验的 SaaS 产品经理"。越具体，AI 的输出就越聚焦。

You are a senior management consultant with 20 years of experience in business strategy and organizational transformation. Your responsibilities: - Analyze business challenges - Recommend data-driven solutions - Identify implementation risks Your task: [Specific task]
❌ 模糊角色
"Act like an expert and help me."
✅ 精准角色
"You are a senior product manager with 10 years of experience in SaaS companies. Your task is to..."
5
Prefilling 预填充响应
▼

直接帮 AI "开头"——告诉它输出的前几行应该长什么样，后面它就会自动跟着你的格式走。

// 预填充结构化分析 Claude's response should start: "Here's my analysis of this market opportunity: Market Size: [Analysis] Growth Potential: [Analysis] Competitive Landscape: [Analysis]"

核心价值：确保格式、设定语气、引导推理方向。相当于"锚定"了 AI 的输出起点。

6
Prompt Chaining 提示词链
▼

把复杂任务拆成多个串行的子 Prompt，前一步的输出作为后一步的输入。像流水线一样处理。

Prompt 1: Extract 提取信息 ↓ Output 1: Structured JSON ↓ Prompt 2: Analyze 分析数据 ↓ Output 2: Analysis report ↓ Prompt 3: Synthesize 生成摘要 ↓ Final: Executive summary

什么时候用：任务涉及"先提取、再分析、最后总结"这种多步骤逻辑时。每一步都能独立验证，出错了只需要重跑一步。

7
Context Management 上下文管理
▼

AI 的上下文窗口是有限的。塞太多信息，重点反而被淹没；信息太少，AI 又会瞎猜。关键是找到平衡点。

// 渐进式披露（Progressive Disclosure） Level 1: High-level overview ← 先给大方向 Level 2: Key components ← 再给关键组件 Level 3: Specific details ← 最后给细节 // 条件信息 If [condition], include [information] Else, skip [information]

实操建议：只放必要上下文、层级化组织、用引用代替全文、先概述后细节。

八大常见故障 + 修复方案（可折叠）

Prompt 不好使？对号入座找解法 👇

!
输出不一致（同一个 Prompt 每次结果不同）
▼

根因：指令含糊、缺少约束、示例不足、没定义成功标准

❌ BEFORE
"Summarize this article"
✅ AFTER
"Summarize this article in exactly 3 bullet points, each 1-2 sentences. Focus on key findings and implications."
!
AI 幻觉（自信地编造虚假信息）
▼

根因：Prompt 鼓励推测、缺乏事实锚定、上下文不足

❌ BEFORE
"What will happen to the market next year?"
✅ AFTER
"Based on current market data, what are 3 possible scenarios for next year? For each, explain your reasoning and note your confidence level (high/medium/low)."

杀手级技巧：让 AI 自问"What don't you know?"——强制它暴露盲区。

!
回答空泛（说了等于没说）
▼

根因：Prompt 太笼统、没给上下文、目标不明确

❌ BEFORE
"How can I improve my business?"
✅ AFTER
"I run a SaaS company with $2M ARR. We're losing customers to competitors. What are 3 specific strategies to improve retention? For each, explain implementation steps and expected impact."
!
输出太长或太短
▼

修复：明确指定字数/句数、定义范围、给出格式模板。

❌ BEFORE
"Explain machine learning"
✅ AFTER
"Explain machine learning in 2-3 paragraphs for someone with no technical background. Focus on practical applications, not theory."
!
格式不对 · AI 拒绝回答 · Prompt 太长 · 无法泛化
▼
  • 格式不对：给出精确的格式示例（JSON、表格等），用 XML 标签结构化
  • AI 拒绝：说明合法用途、提供上下文、换个问法
  • Prompt 太长：删除不必要的上下文、用引用代替全文、渐进式披露
  • 无法泛化：用变量替换硬编码值、处理多种输入格式、加入容错机制
反模式 vs 正确姿势

三组经典对比

❌ 模糊链式
"Analyze this, then summarize it, then give me insights."
✅ 清晰链式
"Step 1: Extract key metrics from the data
Step 2: Compare to industry benchmarks
Step 3: Identify top 3 opportunities
Step 4: Recommend prioritized actions"
❌ 模糊格式
"Give me the results in a nice format."
✅ 精确格式
"Format as a table with columns: Metric, Current, Target, Gap"
❌ 低质量 Prompt
"Analyze this data and tell me what you think about it. Make it good."
✅ 高质量 Prompt
"Analyze this sales data to identify:
1. Top 3 performing products (by revenue)
2. Seasonal trends (month-over-month changes)
3. Customer segments with highest LTV

Format: Executive summary + Key metrics table + Actionable recommendations"

Prompt 质量检查清单

部署前必查 10 项

  • 目标是否 crystal clear？
  • 指令是否足够具体（没有"Make it good"之类的废话）？
  • 输出格式是否明确指定？
  • 是否提供了示例？
  • 边界情况是否有处理？
  • 是否用多种输入测试过？
  • 输出一致性是否够高？
  • Token 用量是否优化过？
  • 错误处理逻辑是否清晰？
  • 文档是否完整（别人看得懂）？
实战案例精选（可折叠）

从 10 个案例中精选 3 个最有启发性的 👇

A
案例：XML 结构化营销文案生成器
▼

这个案例把 XML 标签的优势发挥到了极致——把元数据、指令、约束、格式、示例全部用标签隔离，AI 理解起来毫不含糊：

<prompt> <metadata> <purpose>Generate marketing copy for SaaS products</purpose> <target_audience>B2B decision makers</target_audience> </metadata> <constraints> <max_length>150 words</max_length> <tone>Professional but approachable</tone> <avoid>Jargon, hyperbole, false claims</avoid> </constraints> </prompt>
B
案例：Data Analysis Agent 定制指令
▼

完美示范了 Agent 指令该怎么写——角色 + Do's/Don'ts + 输出格式 + 范围边界，四件套齐全：

## Role Expert financial data analyst ## Do's - Verify data sources before analysis - Provide confidence levels for predictions - Highlight assumptions and limitations ## Don'ts - Don't predict beyond 12 months without caveats - Don't present correlation as causation - Don't skip uncertainty quantification ## Output Format 1. Executive Summary (2-3 sentences) 2. Key Findings (bullet points) 3. Detailed Analysis 4. Limitations and Caveats 5. Recommendations

关键学习：Don'ts 比 Do's 更重要。定义好不该做什么，AI 的犯错空间就小很多。

C
案例：Prompt 测试框架
▼

这个测试框架是整个 Skill 最被低估的部分。大多数人写完 Prompt 就直接用了，从不做系统性测试。

## Test Case 1: Happy Path Input: [Standard, well-formed input] Expected: [Specific output] Criteria: [Measurable] ## Test Case 2: Ambiguous Input Input: [Unclear input] Expected: [Should ask for clarification] ## Test Case 3: Error Handling Input: [Invalid input] Expected: [Helpful error message] ## Regression Test Input: [Previous failing case] Expected: [Now handles correctly]

辣评：如果每个用 AI 的人都用这个框架测一遍自己的 Prompt，AI 输出质量全球平均能提高 3 倍。

3.5 星 — 教科书级方法论，但缺少灵魂

👍 优点

  • 知识体系完整到令人发指。7 大领域 × 8 种能力 × 10 个案例 × 8 种故障诊断 × 质量检查清单 × 学习路径分级（初/中/高）。作为"提示词工程参考手册"堪称满分。如果 Prompt Engineering 有考试，抱着这个 Skill 复习就够了。
  • Before/After 对比是杀手级内容。每个反模式都有"烂 Prompt → 好 Prompt"的对比，一目了然。这比空讲理论有用 100 倍。特别是"Analyze this data and tell me what you think about it. Make it good."这种 Before 示例，太真实了——90%的人就是这么写的。
  • 故障排查手册极其实用。8 种常见问题 + 根因分析 + 具体修复方案 + Quick Reference 表格，调试工作流 5 步法。这是唯一一个我见过的、把 Prompt Debugging 系统化的 Skill。
  • 文件组织堪称 Skill 开发教范。SKILL.md → CLAUDE.md → INDEX.md → docs/ → examples/，层次清晰、渐进式披露做得教科书级。自己就是自己方法论的最佳实践。

👎 缺点

  • 严重的"维基百科综合征"。通篇在描述"什么是 CoT"、"什么是 Few-Shot"这些基础概念，但实战中的 tricky 部分几乎没涉及。比如：CoT 什么时候反而会降低性能？Few-Shot 的例子选错了会怎样？XML 标签嵌套太深 AI 会乱吗？这些真正用起来才会遇到的坑，一个都没讲。
  • 零自动化。号称是提示词工程"专家"，但连一个自动分析 Prompt 质量的脚本都没有。Content Repurposer 好歹有 7 个 shell 脚本（虽然是 mock），ArXiv Reader 有完整的 Python 流水线。这个 Skill 纯靠 AI 读文档后"内化"知识——效果完全取决于 AI 当天的发挥。
  • 内容高度重复。SKILL.md、CLAUDE.md、README.md、INDEX.md、SUMMARY.md、START_HERE.md 这 6 个文件，核心内容重叠率超过 60%。同样的"7 大领域"、"8 种能力"、"9 个用例"在不同文件里反复出现。这不是"渐进式披露"，是复制粘贴。真正不重复的干货集中在 BEST_PRACTICES.md、TECHNIQUES.md 和 EXAMPLES.md 三个文件里。
  • 完全以 Claude 为中心。所有技术讲解、所有示例都针对 Claude/Anthropic 生态。但 Prompt Engineering 不是只有 Claude 一家——GPT 对 XML 标签的响应就不如 Claude、Prefilling 在 OpenAI API 里写法完全不同、不同模型的 CoT 触发方式也有差异。这个"专家"的知识面太窄了。
  • 中文场景缺失。所有示例都是英文。中文 Prompt 的特殊挑战（比如分词歧义、语序灵活性、敬语影响语气控制）零涉及。

💡 谁最该用？

AI 初学者和中级用户——如果你还在写"帮我做 XXX"这种 Prompt，这个 Skill 能让你的提示词水平从 20 分直接跳到 70 分。TECHNIQUES.md 和 EXAMPLES.md 值得反复看。

Skill/Agent 开发者——BEST_PRACTICES.md 里关于 Skill 文件结构、命名规范、渐进式披露的建议确实很实用。可以当作 Skill 开发的 style guide。

高级用户请绕行——如果你已经能熟练使用 CoT + XML + Few-Shot 组合拳，这个 Skill 不会给你带来任何新东西。它在"深度"这个维度上严重不足。
